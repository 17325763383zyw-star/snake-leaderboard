- [ ] 游戏能正常开始和暂停

- [ ] 音效开关默认为开启状态

### 5. 兼容性测试
- [ ] Chrome 浏览器测试

- [ ] 移动设备测试（可选）

## 🧪 测试步骤

### 步骤1：本地测试
1. 在浏览器中打开 `index.html`
2. 检查页面布局是否正常
3. 验证音效开关显示正确

### 步骤2：游戏功能测试
1. 点击"开始游戏"按钮
2. 观察背景音乐是否开始播放
3. 控制蛇吃到第一个果子
4. 观察是否有以下效果：
   - 听到和弦音效
   - 看到粒子爆炸
   - 看到光环扩散
   - 看到 "+10" 飘字

### 步骤3：音效控制测试
1. 取消音效开关
2. 吃到果子，确认没有音效
3. 重新开启音效开关
4. 吃到果子，确认音效恢复

### 步骤4：持久化测试
1. 设置音效开关为关闭状态
2. 刷新页面
3. 确认开关状态保持为关闭

### 步骤5：性能测试
1. 玩游戏几分钟，吃到多个果子
2. 观察游戏是否保持流畅
3. 检查是否有内存泄漏（浏览器开发者工具）

## 🚀 部署测试

### Vercel 部署后测试
1. 确认 Vercel 部署成功

3. 重复上述功能测试
4. 特别检查音效文件是否能正常加载

## 🐛 常见问题

### 音效无法播放
- **原因**：浏览器自动播放策略
- **解决**：确保用户先与页面交互（点击按钮）

### 特效不显示
- **原因**：容器元素未找到
- **解决**：检查 `.relative` 容器是否存在

### 性能问题
- **原因**：粒子元素未正确清理
- **解决**：检查 `setTimeout` 清理逻辑

## 📊 性能基准

- 游戏帧率：60 FPS
- 音效延迟：< 50ms
- 特效动画：800ms 内完成
- 内存占用：< 50MB
